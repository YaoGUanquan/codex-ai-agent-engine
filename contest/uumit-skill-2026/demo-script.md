# Codex AI Agent Engine Demo Script

## 30 秒介绍

大家好，我提交的作品是 Codex AI Agent Engine。它的目标是让 Codex 不只是回答问题，而是像一个可协作的工程团队一样工作：先澄清需求，再生成计划，然后安全地修改代码，最后审查、验证、交接并沉淀经验。它特别适合真实项目里的复杂开发任务，因为它会遵守仓库规则、保护已有改动、记录过程文档，并输出可追踪的交付证据。

## 2 分钟演示流程

1. 用户输入一个模糊需求，例如“帮我给后台加一个权限校验”。
2. 使用 `ae-brainstorm` 澄清业务目标、边界条件和验收标准。
3. 使用 `ae-plan` 生成实现步骤、影响范围和验证计划。
4. 使用 `ae-work` 按计划修改代码，并在动手前检查 Git 状态和项目规则。
5. 使用 `ae-review` 检查潜在缺陷、遗漏测试和接口风险。
6. 使用 `ae-handoff` 或 `ae-save-experience` 把结果沉淀，方便下次继续。

## 评委视角价值

这个 Skill 的创新点不是“再做一个提示词”，而是把成熟工程实践产品化成可复用 Agent 工作流。它解决的是 AI 编程里最常见的问题：上下文丢失、过程不可追踪、改动不可验证、经验无法复用。

## 可展示命令

```text
$ae-lfg
我想给项目增加一个导出功能，请从需求澄清到实现验证完整推进。
```

```text
$ae-review
请审查当前改动，优先指出可能导致线上回归的问题。
```

```text
$ae-save-experience
把这次权限校验修复的经验沉淀为项目长期记忆。
```
