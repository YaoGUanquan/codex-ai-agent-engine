# AE Doc Structure

Turn unstructured text into durable structured documents.

## Workflow

1. Read the source material and identify the target document type.
2. Extract facts, decisions, open questions, risks, tasks, owners, and validation signals.
3. Choose a structure that fits the content instead of forcing all sections.
4. Keep uncertain items under open questions.
5. Use repository-relative paths when writing project artifacts.

## Common Outputs

- Requirements: problem, goals, non-goals, requirements, acceptance criteria.
- Plan: scope, decisions, implementation units, validation, rollback.
- Handoff: current state, changed files, commands, blockers, next steps.
- Checklist: ordered verification or operational steps.
