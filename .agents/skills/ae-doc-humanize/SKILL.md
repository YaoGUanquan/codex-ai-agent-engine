# AE Doc Humanize

Rewrite documents for human readability while preserving facts.

## Workflow

1. Read the source document or supplied text completely.
2. Identify intended audience and whether tone should be operational, executive, tutorial, or handoff.
3. Rewrite for clarity, flow, and scannability.
4. Preserve facts, commands, paths, API names, dates, and decisions.
5. Call out any ambiguous or unsupported claim instead of filling it in silently.

## Rules

- Do not change technical meaning.
- Keep executable commands and file paths exact.
- Do not add marketing language unless requested.
